Anti_depression 项目文件说明（简版）
=================================

一、启动相关
---------------------------------
1) main.py
   - 项目主入口。
   - 负责初始化各模块，启动学生端主界面。

2) start_project.bat
   - 普通启动脚本（会显示控制台窗口）。
   - 适合调试时查看报错日志。

3) start_project_silent.bat
   - 静默启动脚本（后台启动，不显示终端窗口）。
   - 适合日常直接使用。


二、界面层（UI）
---------------------------------
4) ui_main.py
   - 学生端主界面。
   - 包含：实时监测页、波形矩阵页、心流防御页、系统终端页等。
   - 负责接收摄像头/生理/Mock 数据并渲染组件。
   - 负责 AI 干预触发、社交电量显示、按钮交互逻辑。

5) ui_login.py
   - 登录界面（如果启用登录流程）。

6) ui_parent.py
   - 上层界面容器/父级页面管理（历史结构保留）。


三、视觉与融合算法
---------------------------------
7) local_vision.py
   - 本地视觉分析模块。
   - 功能：表情识别、眼疲劳估计、姿态风险估计（探头/低头）。
   - 输出视觉多维特征给 camera_thread。

8) camera_thread.py
   - 摄像头线程（核心实时数据总线之一）。
   - 采集视频帧并调用 local_vision 分析。
   - 结合 mock_sensors 生理数据做多模态融合。
   - 输出：实时风险、多模态细项、社交电量信号。

9) risk_calculator.py
   - 风险计算规则模块。
   - 提供视觉/HRV/EEG 单项风险与综合融合函数。


四、生理信号链路
---------------------------------
10) physio_thread.py
    - 生理数据线程。
    - 功能：串口读取 ESP32/PPG 数据，提取 BPM、HRV 等特征。
    - 当无串口时可启用伪数据模式用于联调。

11) PPG_Monitor.py
    - 独立 PPG/ECG/HR/HRV 监测脚本（可用于波形算法单独调试）。

12) esp32_receiver.py
    - ESP32 数据接收与转发相关脚本（串口/接口桥接）。

13) web_server.py
    - 本地服务层（用于提供接口数据，如心率 latest 等）。


五、模拟与测试
---------------------------------
14) cyber_mock.py
    - UI 组件 + Mock 模拟线程。
    - 包含：SocialBatteryWidget、PredictionCurveWidget、MockWaveformThread。
    - 为无硬件环境提供四路波形和仪表盘模拟数据。

15) mock_sensors.py
    - 模拟硬件传感器数据（HRV/EEG）生成器。
    - 用于无真实硬件时维持融合链路可运行。

16) test_fusion.py
    - 融合逻辑测试脚本。

17) debug_api .py
    - API 调试脚本（文件名含空格，建议后续重命名）。

18) api_client.py
    - API 调用封装或客户端测试逻辑。

19) read.py / download_echarts.py
    - 工具脚本（读取、下载等辅助功能）。


六、数据与配置
---------------------------------
20) config.py
    - 全局配置中心。
    - 包含阈值、窗口尺寸、风险等级参数、电量策略参数等。

21) recorder.py
    - 风险记录/持久化模块。
    - 负责写入与读取历史风险数据，供图表回放。

22) shadow_cases.json（运行时生成）
    - 影子模式误报样本记录文件（每行一个 JSON）。
    - 用于后续模型微调与误报分析。


七、推荐运行方式
---------------------------------
1) 日常使用：双击 start_project_silent.bat
2) 调试排错：双击 start_project.bat（可看控制台报错）


八、常见问题
---------------------------------
1) 双击静默脚本没反应：
   - 先用 start_project.bat 看是否有依赖缺失。

2) 提示缺少模块（如 serial / mediapipe）：
   - 在项目环境中安装对应依赖后重试。

3) 无硬件时看不到生理变化：
   - 检查 config.py 中是否启用了伪数据模式。


（说明文件可根据后续模块变更继续维护）
